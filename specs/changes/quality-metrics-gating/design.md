# Design: quality-metrics-gating

## Summary
Wave 3 Track H makes the translation-quality subsystem score-driven instead of advisory. CometKiwi QE becomes default-on and scores each translatable element individually; segments below `QE_RESCORE_THRESHOLD` are routed to a re-translation pass. The critique loop gains a non-regression gate that adopts a revision only when its score does not drop below the original, using the QE model inline as the scorer and a length/fluency heuristic when QE is unavailable. The LLM judge moves from one whole-document verdict to per-block scoring, and PDF jobs gain an additive MLLM-as-judge layout score (1-5) on the in-memory rasterised page, reusing the local Gemma infra. Finally `translate_document()` (long-doc path) reaches parity with the short-doc path by delegating per-chunk translation to `translate_texts()`, inheriting terms substitution and the critique gate, and feeding the 50-token chunk overlap in as read-only context. All behaviour changes are reversible through existing env flags.

## Key Decisions

- **Critique gate score source**: the gate scores `(src, draft)` vs `(src, revised)` with the CometKiwi reference-free model inline (`score_blocks(model, [(src,draft),(src,revised)])`) and adopts the revision iff `score(revised) >= score(original)` → rejected alternative: scoring only post-translation (BR-55 pass) — the gate must compare drafts *during* the loop, so the evaluator primitive is reused inline. Consequence: when `CRITIQUE_LOOP_ENABLED` and `QE_ENABLED` are both true the COMET model now loads during translation, not only post-translation (VRAM/latency note below).
- **Critique tie-handling**: `>=` is the non-regression invariant; an exact tie resolves to **keep the original text** (text-level no-op — identical quality, one fewer mutation, deterministic). Net effect: the revised text wins only on strict improvement; equal or worse revisions are discarded → rejected alternative: "last iteration always wins" (current behaviour) which silently regressed quality.
- **No-QE fallback heuristic**: when the QE model is unavailable/disabled the gate falls back to a deterministic scalar = length-ratio sanity (penalise truncation/runaway vs source length band) combined with a fluency proxy (penalise empty output, `[Translation failed|` markers, gross repetition). Adopt revised iff `heuristic(revised) >= heuristic(original)`; same tie rule → rejected alternative: blind last-wins fallback (no guard) and full secondary LLM judge call (too costly per segment).
- **Per-segment QE call shape (4.1)**: one batched call `score_blocks(model, [(e.content, e.translated_content) for translatable e], device) -> List[float]`, positionally zipped back to elements — NOT one `model.predict` per segment. "Per-segment" = one score per element in the returned list. Tests must target `score_blocks` / the post-translate QE hook, not a wrapper (avoids wrong-entry-point tautology) → rejected alternative: per-segment `predict()` calls (orders-of-magnitude slower, same result).
- **Per-block judge — Approach A**: iterate blocks calling the existing single-pair `QualityJudge.evaluate(src, tgt)` per block and collect per-block scores, instead of the current whole-doc join in `_run_judge_loop_impl` → rejected alternative B (call `judge_translation` per block with truncated context): duplicates the prompt path and loses focused context; `evaluate()` is already a per-pair primitive, so A reuses it with no new prompt.
- **MLLM layout scoring interface**: `quality_judge.judge_layout()` receives an **in-memory PIL image**, never a file path. Coupling point is one-directional: the PDF processor rasterises the page (reusing Track B harness / `PDF_RENDER_DPI` pixmap) and calls `judge_layout`; `quality_judge` never imports PDF internals → rejected alternative (pass a file path): forces a disk write, violating the local-image privacy posture (BR-32). The image is sent only to the local Gemma model (D4 isolation), never to a cloud provider — see ADR 0007.
- **`translate_document()` wiring (4.4)**: delegate each chunk's text list to the existing `translate_texts()` entry point (which already performs terms-context, the critique loop + gate, and glossary substitution) instead of the bare `translate_blocks_batch`, then map the returned `tmap` back onto chunk elements → rejected alternative (inline-call critique + `apply_glossary_substitution` separately): the critique logic is inline inside `translate_texts`, not a standalone function, so duplicating it would drift from the short-doc path. Overlap parity: pass the previous chunk's trailing 50-token overlap text as read-only sliding context (same mechanism as `CONTEXT_WINDOW_SEGMENTS`) into the current chunk's prompt — overlap stops being dedup-only.

## Data Flow
Translation produces `translated_content` per element. During translation the critique loop (if enabled) scores draft vs revision inline (QE model or heuristic) and keeps the non-regressing text. Post-translation (BR-55 timing), QE batch-scores all translatable `(src, mt)` pairs → `List[float]` → one `BlockQualityScore` per element on `JobQualityRecord`. Elements with `score < QE_RESCORE_THRESHOLD` are collected and routed through a second translation pass, then re-scored. If `JUDGE_ENABLED`, the judge emits a per-block 高/中/低 score; for PDF jobs the rasterised page additionally yields a 1-5 layout score via the local MLLM. All quality records are additive and parallel to `JobRecord.quality` / `.judge`; consumers that ignore them are unaffected.

## Migration / Rollback
No data migration. Every behaviour is flag-gated: set `QE_ENABLED=false` to restore the prior QE-off baseline (also disables the QE-backed critique gate, which then uses the heuristic), `CRITIQUE_LOOP_ENABLED=0` to drop the gate entirely, and `JUDGE_ENABLED` stays `false` by default so the per-block judge and MLLM layout score are opt-in. `QE_RESCORE_THRESHOLD` is additive with a documented default in the env contract. Rollback is instant via env flags; no contract or schema downgrade is required beyond reverting the default.

## Open Risks
- **VRAM/latency**: `QE_ENABLED` default-on lazy-loads COMET for every job, and the inline critique gate now loads the QE model during translation when both flags are on — concurrent COMET + translation LLM + optional Gemma judge raises peak VRAM. Quantify in `qa-report.md` / `stress-soak-report.md`; gate to default-off if a blocking regression is found.
- **`QE_RESCORE_THRESHOLD` default value**: CometKiwi-da outputs ~0-1 but the scale is model-dependent; the concrete default is owned by the env contract and must be tuned against a small corpus, not guessed in code.
- **MLLM layout privacy**: page images now leave the rasteriser to the local Ollama socket (extends BR-32's in-process-only posture to local-socket). Locked to local Gemma only; must never be routed to a cloud provider — enforced by ADR 0007 and judge D4 isolation.
- **API surface (CER-001)**: contract-reviewer must confirm whether per-block judge / per-segment QE arrays extend `GET /jobs/{id}/quality` or `GET /judge`; if so, update `api-contract.md` and re-run `cdd-kit openapi export`.
