## MODIFIED Requirements

### Requirement: RouteGroup Carries Refine Model
`RouteGroup` SHALL include an optional `refine_model` field that specifies which model to use for the cross-model refinement pass.

#### Scenario: HY-MT group gets Qwen refiner
- **GIVEN** `resolve_route_groups()` resolves targets including Vietnamese, Japanese, or German
- **WHEN** a RouteGroup is created for HY-MT
- **THEN** `RouteGroup.refine_model` SHALL be set to `DEFAULT_MODEL` (Qwen3.5:4b)

#### Scenario: TranslateGemma group gets Qwen refiner
- **GIVEN** `resolve_route_groups()` resolves a Korean target
- **WHEN** a RouteGroup is created for TranslateGemma
- **THEN** `RouteGroup.refine_model` SHALL be set to `DEFAULT_MODEL` (Qwen3.5:4b)

#### Scenario: Qwen group has no refiner
- **GIVEN** `resolve_route_groups()` resolves targets using Qwen (e.g., English)
- **WHEN** a RouteGroup is created for Qwen
- **THEN** `RouteGroup.refine_model` SHALL be `None`

#### Scenario: Manual profile override disables refinement
- **GIVEN** a user-specified profile override (non-auto)
- **WHEN** `create_job()` creates a single RouteGroup from the override
- **THEN** `RouteGroup.refine_model` SHALL be `None`
- **AND** no cross-model refinement SHALL occur

### Requirement: Refine OllamaClient Uses Domain-Aware System Prompt
The refine OllamaClient created in `process_files()` SHALL be initialized with a system prompt generated by `_build_refine_system_prompt(target_language, profile_id)`.

#### Scenario: process_files creates domain-specific refine client
- **GIVEN** `refine_model` is provided and `CROSS_MODEL_REFINEMENT_ENABLED=True`
- **WHEN** `process_files()` creates the refine OllamaClient
- **THEN** it SHALL call `_build_refine_system_prompt(first_target, profile_id)` to get the system prompt
- **AND** construct `OllamaClient(model=refine_model, model_type="general", system_prompt=refine_system_prompt)`

#### Scenario: refine system prompt varies by target language
- **GIVEN** a HY-MT group with targets=["Vietnamese", "Japanese"]
- **WHEN** `process_files()` builds the refine client
- **THEN** the system prompt SHALL be generated for the first target language ("Vietnamese")
- **AND** SHALL describe a Vietnamese manufacturing engineer persona

### Requirement: process_files Passes Refine Model
`process_files()` SHALL accept a `refine_model` parameter and construct a minimal `OllamaClient` for refinement when the parameter is provided and `CROSS_MODEL_REFINEMENT_ENABLED` is True.

#### Scenario: process_files creates refine client
- **GIVEN** `refine_model` is a non-empty string and `CROSS_MODEL_REFINEMENT_ENABLED=True`
- **WHEN** `process_files()` initializes
- **THEN** it SHALL construct a `refine_client = OllamaClient(model=refine_model, model_type="general")` with no system prompt
- **AND** pass `refine_client` to `translate_texts()`

#### Scenario: process_files with no refine model
- **GIVEN** `refine_model=None`
- **WHEN** `process_files()` initializes
- **THEN** `refine_client` SHALL be `None`
- **AND** `translate_texts()` receives `refine_client=None`
