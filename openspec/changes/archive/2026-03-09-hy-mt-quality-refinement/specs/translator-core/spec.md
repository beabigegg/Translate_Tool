## MODIFIED Requirements

### Requirement: Two-Phase VRAM-Safe Cross-Model Refinement
`translate_texts()` SHALL execute primary translation and refinement in two strictly separated phases to prevent OOM on 8GB VRAM hardware.

#### Scenario: Phase 1 — HY-MT translates all targets
- **GIVEN** `translate_texts()` is called with HY-MT as `client` and a non-None `refine_client`
- **WHEN** the function runs
- **THEN** Phase 1 SHALL translate ALL target languages sequentially using `client` (HY-MT)
- **AND** accumulate all results in `tmap` before any refinement begins
- **AND** NOT call `refine_client` during Phase 1

#### Scenario: Phase 2 — Explicit HY-MT unload before Qwen refinement
- **GIVEN** Phase 1 is complete and `CROSS_MODEL_REFINEMENT_ENABLED=True`
- **WHEN** Phase 2 begins
- **THEN** the system SHALL call `client.unload_model()` BEFORE the first call to `refine_client`
- **AND** Qwen SHALL load only after HY-MT is evicted from VRAM
- **AND** Qwen SHALL refine ALL qualifying segments across ALL target languages in one contiguous sweep

#### Scenario: Phase 2 — Segment-batch refinement
- **GIVEN** `CROSS_MODEL_REFINEMENT_ENABLED=True` and `refine_client` is provided
- **WHEN** Phase 2 runs for a target language
- **THEN** for each source text with `len(source_text) >= REFINEMENT_MIN_CHARS`
- **AND** `tmap[(tgt, text)]` exists (i.e., primary translation succeeded)
- **THEN** the system SHALL call `refine_client.refine_translation(source_text, draft, tgt, src_lang)`
- **AND** replace `tmap[(tgt, text)]` with the refined result if `ok=True` and result is non-empty
- **AND** retain the original draft if refinement returns `ok=False`

#### Scenario: Short segments skip refinement
- **GIVEN** `len(source_text) < REFINEMENT_MIN_CHARS`
- **WHEN** Phase 2 runs
- **THEN** the segment SHALL be skipped and its primary translation kept unchanged

#### Scenario: Refinement disabled globally
- **GIVEN** `CROSS_MODEL_REFINEMENT_ENABLED=False`
- **WHEN** `translate_texts()` is called with any `refine_client`
- **THEN** Phase 2 SHALL be skipped entirely; no `client.unload_model()` call
- **AND** primary translations returned unchanged

#### Scenario: No refine_client — no unload
- **GIVEN** `refine_client=None`
- **WHEN** translation completes
- **THEN** Phase 2 SHALL NOT occur and `client.unload_model()` SHALL NOT be called by `translate_texts()`

#### Scenario: Qwen primary — no refinement
- **GIVEN** a Qwen RouteGroup with `refine_model=None`
- **WHEN** `translate_texts()` is called
- **THEN** `refine_client` SHALL be `None` and Phase 2 SHALL NOT occur

### Requirement: Domain-Aware Qwen Refiner System Prompt
The Qwen refine client SHALL receive a domain and language-specific system prompt generated by `_build_refine_system_prompt(target_language, profile_id)`.

#### Scenario: technical_process profile refine system prompt
- **GIVEN** `profile_id="technical_process"` and `target_language="Vietnamese"`
- **WHEN** `_build_refine_system_prompt("Vietnamese", "technical_process")` is called
- **THEN** the returned string SHALL establish a persona as a senior Vietnamese process/manufacturing engineer
- **AND** SHALL instruct to cross-reference the source for terminology correctness
- **AND** SHALL instruct to correct unnatural literal renderings to proper industrial terms
- **AND** SHALL instruct SOP-register formality
- **AND** SHALL prohibit explanations and dialogue ("output only the corrected Vietnamese")

#### Scenario: Refine user prompt format uses labeled sections
- **GIVEN** source text and HY-MT draft are available
- **WHEN** `_build_refine_prompt(source_text, draft, target_language, source_language)` builds the user prompt
- **THEN** the prompt SHALL label the source text as `[SOURCE]:` and the draft as `[DRAFT]:`
- **AND** end with a completion cue: `Corrected {target_language}:`

#### Scenario: Unknown profile falls back to generic manufacturing persona
- **GIVEN** `profile_id` not in the refine system prompt mapping
- **WHEN** `_build_refine_system_prompt()` is called
- **THEN** it SHALL return a generic manufacturing/industrial engineering persona for the given target_language
- **AND** include the same 4 rules (terminology, literal correction, SOP register, output-only)

### Requirement: Refinement Does Not Affect Cache
Refined translation output SHALL NOT be written to the translation cache.

#### Scenario: Cache write for primary translation only
- **GIVEN** cross-model refinement is enabled and a segment is refined by Qwen after HY-MT primary translation
- **WHEN** cache writes occur
- **THEN** only the HY-MT primary translation SHALL be written to the cache
- **AND** the Qwen-refined result SHALL NOT be written to the cache

#### Scenario: Cache hit bypasses refinement
- **GIVEN** a cache hit is served for a segment
- **WHEN** Phase 2 runs
- **THEN** cached segments SHALL NOT be sent to the refine client
